SESSION LIFECYCLE (4 events)
========================================================================

1. mcp_session_started
   - When: New session created
   - Props: session_id, agent_id

2. mcp_session_ended
   - When: Session terminated or timed out
   - Props: session_id, agent_id, duration_seconds, reason

3. mcp_sse_connected
   - When: Client opens SSE stream
   - Props: session_id, agent_id

4. [Session timeout tracking embedded in cleanup interval]
   - Tracks all terminations with reason='timeout'

========================================================================
PROTOCOL-LEVEL (3 events)
========================================================================

5. mcp_tools_listed
   - When: Client calls tools/list
   - Props: session_id, agent_id, tool_count

6. mcp_tool_called
   - When: Every tool execution (success or failure)
   - Props: session_id, agent_id, tool_name, success, latency_ms, error_type

7. mcp_tool_error
   - When: Tool fails
   - Props: session_id, agent_id, tool_name, error_type, latency_ms

========================================================================
ERROR TYPES
========================================================================

When mcp_tool_error or mcp_tool_called (success=false) occurs,
error_type is one of:

- tool_error: Tool returned error object
- rate_limited: Rate limit exceeded
- auth_failed: Authentication failed
- not_found: Resource not found
- validation: Input validation failed
- server_error: Upstream server error

========================================================================
IMPLEMENTATION PATTERNS
========================================================================

1. Session Tracking
   Location: mcp-remote.ts createSession(), cleanup interval, DELETE endpoint
   Pattern: Track lifecycle events with duration/reason

2. Protocol Tracking
   Location: mcp-remote.ts tools/list and tools/call cases
   Pattern: Wrap execution with startTime, measure latency, track errors

3. Tool-Level Tracking
   Location: mcp-tools.ts each tool case
   Pattern: After successful fetch, extract safe data and track

4. Error Categorization
   Location: mcp-remote.ts tools/call case
   Pattern: Parse error messages to categorize by type

========================================================================
PRIVACY SAFEGUARDS
========================================================================

NOT TRACKED:
- API keys, tokens, secrets
- Full request/response bodies
- Personal information
- Callback URLs/secrets
- Job descriptions/listing content

TRACKED SAFELY:
- Event names and counts
- Non-sensitive parameters
- Numeric aggregates
- Boolean flags
